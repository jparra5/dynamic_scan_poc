Описание
===========
Этот файл ZIP содержит файлы, необходимые для настройки AppScan Presence в сети.

Если приложение недоступно в Интернете, то необходимо создать AppScan Presence с доступом к
приложению и Интернету, к которому может подключиться Application Security on Cloud. Поддерживаются соединения через прокси-сервер.


Предварительные требования
=============
Требования для Windows Server, Windows Client и Linux доступны по следующему адресу:
- http://www.oracle.com/technetwork/java/javase/certconfig-2095354.html
AppScan Presence поддерживает только 64-разрядные версии.


============
1. Распакуйте файлы AppScan Presence и сохраните их в системе, которая находится в той же
сети, что и сайт. Система должна иметь доступ к сайту и Интернету.

2. Если Presence подключается к приложению или Интернету через прокси-сервер, настройте его в соответствии с инструкциями из следующего раздела.

3. В корневой папке Presence найдите и запустите файл startPresence.sh (Linux) или startPresence.vbs (Windows).

4. В случае настройки сканирования частного сайта включите подключение Application Security on Cloud к приложению по защищенному соединению.
	
	
Настройте параметры прокси-сервера Presence
=================================
Если Presence должен подключаться к приложению через прокси-сервер (внутренний прокси-сервер) или Интернет (исходящий прокси-сервер), то для настройки параметров выполните следующие действия:

1. В корневой папке окружения найдите config.properties и откройте его в текстовом редакторе.

2. Добавьте нужные параметры (удалите "#" в начале каждой соответствующей строки) и сохранить файл. (Обратите внимание, что для "Host" можно указать имя или IP-адрес).

Например, для настройки внутреннего прокси с IP 192.168.1.100 и портом 3128 измените:

#internalProxyHost = 
#internalProxyPort =

на

internalProxyHost = 192.168.1.100 
internalProxyPort = 3128

3. Запустите Presence с помощью файла startPresence.sh (Linux) или startPresence.vbs (Windows).
